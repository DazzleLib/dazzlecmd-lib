**Target versions:** dazzlecmd-lib `0.10.8-alpha` (`1ce1b13`/`2b4e129`), dazzlecmd `0.11.24-alpha`..`0.11.27-alpha` (`b1ac6ec`..`c512f83`), branches `fqcn-unification`.

## What this is
The post-Slice-3 group: the one-token `=` assignment marker (`dz level=kit`), the `dz -h`/`dz prop -h`/`dz level -h` help reshapes, and `docs/grammar.md`. Every write uses a temp `DAZZLECMD_CONFIG` (set it first: `%TEMP%\eq_check.json` / `$env:TEMP\eq_check.json` / `/tmp/eq_check.json`; unset at the end).

## High-Value Verification (~4 minutes)

### HV.1 — the `=` marker (the new grammar surface)
- [ ] `dz level=kit` → sets + echo shows the explicit form; `dz level` → `kit`.
- [ ] `dz .note="some words"` → multi-word value, ONE token, no quoting error; `dz .note` reads it back.
- [ ] `dz :.kit.channels.verbosity=-3` → −3, no `--` needed.
- [ ] `dz .x=` → sets the EMPTY string (works even on PowerShell 5.1); `dz prop get .x` shows it; `dz prop delete .x` removes it.
- [ ] `dz .note=a=b` → stores `a=b` (RHS opaque past the first `=`).
- [ ] `dz name=x` (unregistered bare word) → falls through to the normal invalid-choice error — NO store write.
- [ ] `dz level=bogus` → the invalid-level error, exit 2 — identical to `dz level bogus`.

### HV.2 — help surfaces
- [ ] `dz -h`: `prop` is NOT a top-level row; `meta` row reads "The internals namespace [dz:.meta]: prop, level, use, reset"; `level` row shows its canonical home.
- [ ] `dz prop -h`: description states canonical `dz:.meta:prop`, the path forms, the sugar, and delete-has-no-sugar.
- [ ] `dz level -h`: the rung ladder legend (fiber → supra, one-liner each) + the gentle-default doctrine.
- [ ] `dz level kit` (space form): sets AND the echo teaches `dz level=kit`.

### HV.3 — nothing regressed
- [ ] `dz list` / `dz kit -h` / `dz :list` byte-familiar; `dz prop add/get/delete` and the space sugar (`dz .y hi`) unchanged; byte-gate 10/10.
